# 《横版 / 俯视切换实验游戏》美术风格与概念方向

版本：0.1  
状态：美术讨论稿  
关联文档：[GDD_视角切换玩法方案.md](./GDD_视角切换玩法方案.md)

## 1. 美术目标

这个玩法很依赖玩家对空间的理解。美术方向需要帮助玩家看懂两件事：

1. 这是同一个空间。
2. 换一个视角后，空间规则发生了变化。

所以美术不是单纯做氛围，也要承担机制表达。玩家需要通过场景层级、材质、轮廓、影子、色彩和动画，判断哪些东西能跳、能推、能挡、能砸、能绕。

## 2. 方向一：2D 纸片剧场 / 立体书

### 概念

关卡像一座可以被重新观看的纸片舞台。横版是舞台侧景，俯视是布景平面。箱子、桥、墙、机关都像可以重新摆放的舞台道具。

切换视角时，可以做成翻页、折叠、转台、纸层展开，或者像舞台布景被重新拉开。

### 优点

- 和视角切换机制贴得很近。
- 横版和俯视之间的空间差异容易被玩家接受。
- 对小团队相对友好。
- 道具、机关、平台都可以用“舞台道具”的逻辑统一起来。

### 风险

- 层级和碰撞要画得非常清楚。
- 纸片风格如果太轻，追逐战和 Boss 的压迫感需要靠音效、镜头、色彩和节奏补强。
- 需要避免画面太“装饰化”，影响玩家判断可交互物。

### 适合气质

- 奇幻。
- 轻叙事。
- 机关童话。
- 带一点手工感的冒险。

### 可讨论关键词

- 纸片舞台。
- 立体书。
- 剪纸机关。
- 手工模型。
- 翻页式空间变化。

## 3. 方向二：2D 工业遗迹 / 实验设施

### 概念

世界像一个空间实验设施，视角切换是装置对空间进行扫描和重构。横版展示管道、升降机、断桥和压机；俯视展示走廊、闸门、巡逻机器人和压力板。

### 优点

- 机关逻辑自然。
- 很适合箱子、开关、传送带、激光、压机、机器人。
- 规则感强，玩家容易相信这些机关为什么会这样运作。
- 追逐战和 Boss 可以做出明确的压迫感。

### 风险

- 容易偏冷，需要靠角色、剧情和场景细节增加情绪。
- 如果界面和提示太多，可能显得像纯解谜工具。
- 需要避免灰、蓝、黑占比过高导致画面单调。

### 适合气质

- 实验。
- 科幻。
- 废弃设施。
- 机械谜题。

### 可讨论关键词

- 空间实验室。
- 废弃工厂。
- 维度扫描。
- 机械重构。
- 失控设施。

## 4. 方向三：2D 手绘异空间

### 概念

横版像绘本剖面，俯视像地图或记忆平面。视角切换像世界被重新讲述，或者像主角在不同记忆层之间观察同一个地方。

### 优点

- 空间不合理的地方更容易被接受。
- 适合做情绪和叙事。
- 可以通过色彩区分两个视角的规则。
- 适合把能力和角色心理、梦境、记忆联系起来。

### 风险

- 规则可读性需要特别控制。
- 如果表现过于自由，玩家会难以判断什么能交互。
- 需要提前统一交互物、危险物和背景物的视觉语言。

### 适合气质

- 梦境。
- 记忆。
- 童话。
- 神秘探索。

### 可讨论关键词

- 梦境剖面。
- 记忆地图。
- 手绘空间。
- 两层现实。
- 情绪化场景。

## 5. 方向四：3D 低多边形正交 2.5D

### 概念

用 3D 模型搭建场景，但玩法仍以 2D 规则为核心。横版使用侧向正交相机，俯视使用顶向正交相机。

### 优点

- 视角切换直观。
- 同一个物体从侧面和顶部观看的差异天然成立。
- 更容易做出第一眼卖点。
- 适合后续扩展相机转动、模型重构和空间折叠表现。

### 风险

- 技术复杂度更高。
- 镜头遮挡、碰撞映射、物体层级都需要严格处理。
- 场景资产需要从两个角度都能读懂。
- 如果模型信息太多，俯视状态下可能不好判断路径。

### 适合气质

- 微缩冒险。
- 风格化动作解谜。
- 可爱但不幼稚的模型世界。

### 可讨论关键词

- 正交 2.5D。
- 低多边形模型。
- 侧视 / 顶视相机。
- 模型盒。
- 风格化机关。

## 6. 方向五：3D 微缩模型 / 桌面沙盘

### 概念

关卡是一座摆在桌面上的小型机关盒。横版像从侧面看机关结构，俯视像从上方看迷宫布局。切换时相机可以围绕模型转动，让玩家感觉自己正在观察同一个实体空间。

### 优点

- 和“同一空间，不同视角”的玩法很契合。
- 切换镜头时可以做出很强的展示效果。
- 适合让玩家观察完整关卡结构。
- 推箱、机关、开关、桥梁、轨道都可以做成可读性很高的实体小道具。

### 风险

- 美术资产量可能更高。
- 需要控制相机转动时的信息量，避免玩家晕或迷失。
- 如果场景太精细，机制可读性会下降。
- Boss 和追逐战需要处理好镜头尺度，否则容易显得小打小闹。

### 适合气质

- 机关盒。
- 桌游感。
- 模型玩具。
- 精致小关卡。

### 可讨论关键词

- 桌面沙盘。
- 机关盒。
- 微缩世界。
- 模型玩具。
- 可旋转关卡。

## 7. 方向六：3D 超现实建筑 / 空间折叠

### 概念

世界可以设定为会被视角重新折叠的建筑、梦境或记忆结构。切换视角时，不只是相机变化，空间本身也像被重新组织。

### 优点

- 概念感强。
- 视觉差异明显。
- 适合做独特卖点和叙事隐喻。
- 可以把主角能力、世界观和关卡规则绑定得很紧。

### 风险

- 玩法可读性风险最大。
- 必须严格用颜色、材质、轮廓、动画区分可交互内容。
- 如果美术太抽象，玩家可能无法建立稳定规则。
- 关卡设计和美术表达需要高度同步。

### 适合气质

- 艺术化。
- 悬疑。
- 心理空间。
- 叙事驱动。

### 可讨论关键词

- 空间折叠。
- 不稳定建筑。
- 梦境结构。
- 记忆迷宫。
- 透视错觉。

## 8. 当前推荐方向

如果团队规模较小，优先推荐：

- 2D 纸片剧场 / 立体书。
- 2D 工业遗迹 / 实验设施。

如果团队有能力处理 3D 场景和相机切换，推荐：

- 3D 微缩模型 / 桌面沙盘。
- 3D 低多边形正交 2.5D。

最稳的组合：

- 玩法：动作解谜为主，轻战斗为辅。
- 美术：纸片剧场或微缩模型。
- 表现：切换时像舞台转场、空间折叠或模型盒旋转。

这两个方向最容易让玩家理解“同一个空间在不同角度下有不同规则”，也能支持推物、机关、敌人、追逐战和 Boss。

## 9. 美术需要提前统一的规则

### 9.1 可交互物

建议统一以下内容：

- 可推动物体的轮廓、材质或边缘光。
- 可踩平台的厚度和接触面表现。
- 可破坏物的裂纹、颜色或结构特征。
- 可触发机关的按钮、压力板、拉杆外形。
- 只在某个视角有效的物体提示。

### 9.2 高度与落点

横版和俯视之间最容易出问题的是高度信息。

需要考虑：

- 高处物体在俯视中是否显示影子。
- 坠落物是否显示落点范围。
- 高低层是否使用不同亮度或材质。
- 玩家能否一眼看出哪里能跳、哪里只是背景。

### 9.3 危险物

危险物需要从普通背景中跳出来。

建议统一：

- 尖刺、激光、压机、滚石、坠物的颜色语言。
- 即将触发时的预警动画。
- 伤害范围的地面提示。
- 横版和俯视下是否使用同一套危险提示。

### 9.4 视角切换表现

切换表现是这个项目的重要卖点之一。

可选方向：

- 纸片翻折。
- 舞台转台。
- 空间扫描线。
- 模型盒旋转。
- 建筑折叠。
- 记忆重组。

表现需要和最终美术概念一致，不建议单独做成通用特效。

## 10. 待美术讨论问题

- 最终更倾向 2D、2.5D 还是 3D？
- 两个视角是否使用明显不同的色彩方案？
- 切换视角时，是相机变化、世界变化，还是两者都有？
- 可交互物是否统一高亮？
- 高度信息在俯视状态下怎么表达？
- 追逐战和 Boss 是否需要更强烈的视觉压迫感？
- 主角能力的视觉来源是什么：科技、魔法、梦境、机关，还是世界规则？
- 场景是否需要从两个视角都保持好看，还是优先保证玩法可读性？
